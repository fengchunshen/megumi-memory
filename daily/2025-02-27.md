# 2025-02-27

## Claude Code CLI 配置经验

### 环境搭建
- 安装：`npm install -g @anthropic-ai/claude-code`
- 版本：2.1.62
- 配置文件：`~/.claude/settings.json`

### 中转站 API 配置
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "你的key",
    "ANTHROPIC_BASE_URL": "https://code.newcli.com/claude/droid"
  },
  "language": "中文",
  "model": "opus"
}
```

### 踩坑记录
1. root 用户不能用 `--dangerously-skip-permissions`（安全限制）
2. 解决方案：用非 root 用户（lighthouse）来跑 Claude Code
3. nvm 装的 node 在 /root 下，其他用户没权限访问
4. 解决：把 node 实体文件复制到 `/usr/local/bin/node`（不能是软链接）
5. Claude Code 模块也要复制到公共路径 `/usr/local/lib/node_modules/@anthropic-ai/claude-code/`
6. 创建 wrapper 脚本 `/usr/local/bin/claude-code`

### 调用方式
```bash
sudo -u lighthouse bash -c 'cd /工作目录 && ANTHROPIC_API_KEY="key" ANTHROPIC_BASE_URL="url" /usr/local/bin/claude-code --dangerously-skip-permissions -p "任务描述"'
```

### `-p` 模式说明
- `claude -p "问题"` — 单次问答，不调用工具（print 模式）
- `claude --dangerously-skip-permissions -p "任务"` — 可以调用工具创建/修改文件

## OpenClaw 多智能体配置

### 创建智能体
```bash
openclaw agents add "senior-dev" --model "anthropic/claude-opus-4-6" --workspace ~/.openclaw/workspace-senior-dev --non-interactive
openclaw agents set-identity --agent senior-dev --name "高级开发" --emoji "🔧"
```

### 子智能体权限配置
- 在 `agents.list[].subagents.allowAgents` 中添加允许调用的智能体 ID
- 跨智能体历史查看：`tools.agentToAgent.enabled: true`
- 会话可见性：`tools.sessions.visibility: "all"`

### 嵌套子智能体
- 默认 `maxSpawnDepth: 1`，子智能体不能再 spawn
- 设置 `maxSpawnDepth: 2` 可实现编排链路：main → 编排者 → 执行者
